# WorldOS 代码评审规则

## 评审流程

### 1. 提交时机
- 每个任务完成前必须提交评审
- 评审通过后才能进入下一任务

### 2. 评审人
| 任务 | 评审人 |
|------|--------|
| W2 数据采集 | 龙六 → 龙五 |
| W3 前端界面 | 龙六 → 龙五 |
| W4 预警逻辑 | 龙六 → 龙五 |
| W6 测试 | 龙五 → 龙六 |

### 3. 评审检查清单

#### 功能检查
- [ ] 数据采集：24个指标是否都有数据源？
- [ ] 前端界面：6宫格布局是否符合设计？
- [ ] 预警逻辑：三级预警条件是否正确？

#### 代码质量
- [ ] 无TypeScript编译错误
- [ ] 无console.error
- [ ] 变量命名清晰
- [ ] 关键逻辑有注释

#### 部署检查
- [ ] 本地build通过
- [ ] 页面能正常加载
- [ ] 数据能正常显示

### 4. 评审输出格式

```
## 评审结果

### 通过/不通过

### 问题列表
1. 问题描述

### 修改建议
- 建议
```

### 5. 不通过处理
- 立即修改
- 重新提交评审
- 记录到问题库

---

## 评审时间要求

- 评审人在收到评审请求后15分钟内响应
- 评审不通过需说明具体问题
